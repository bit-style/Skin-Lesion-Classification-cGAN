# Evaluating Generative AI Data Augmentation for Skin Cancer Classification

**IT504 — Selected Topics | King Saud University**
**Author:** Abdulrahman Alqarni (446100634)

This project evaluates the impact of **generative AI data augmentation** on medical image
classification. Using the [HAM10000](https://www.kaggle.com/datasets/kmader/skin-cancer-mnist-ham10000)
skin lesion dataset, a **Conditional GAN (cGAN)** is trained to synthesize images for the
under-represented (minority) classes. Four classical ML models are then trained twice — once on
the original data (**Model-A**) and once on the GAN-augmented data (**Model-B**) — to measure how
augmentation affects classification performance, with particular attention to minority and
medically critical classes.

## Dataset

**HAM10000** ("Human Against Machine with 10000 training images") — 10,015 dermatoscopic images
across seven diagnostic classes. The dataset is heavily imbalanced.

| Class | Description | Images |
|-------|-------------|-------:|
| nv    | Melanocytic nevi | 6,705 |
| mel   | Melanoma | 1,113 |
| bkl   | Benign keratosis-like lesions | 1,099 |
| bcc   | Basal cell carcinoma | 514 |
| akiec | Actinic keratoses / intraepithelial carcinoma | 327 |
| vasc  | Vascular lesions | 142 |
| df    | Dermatofibroma | 115 |

The imbalance is the central problem this project addresses: minority classes such as `df` and
`vasc` are severely under-represented, which causes classifiers to ignore them.

## Methodology

1. **Preprocessing** — images resized to 32×32 RGB; pixel values scaled to [0, 1]; labels encoded.
2. **Stratified split** — 80% train / 20% test. The test set is frozen and used by both models; it
   is **never** augmented (prevents data leakage).
3. **Model-A (baseline)** — SVM, Decision Tree, Random Forest, and Naive Bayes trained on the
   original training set.
4. **Conditional GAN** — a label-conditioned DCGAN trained on the training images, then sampled to
   generate synthetic images for each minority class up to a target count.
5. **Model-B (augmented)** — the same four models trained on the GAN-augmented training set.
6. **Comparison** — both models evaluated on the identical frozen test set using accuracy,
   precision, recall, F1-score, and AUC, plus confusion matrices, ROC curves, and per-class F1.

Only the training data differs between Model-A and Model-B, so any change in performance is
attributable to the augmentation.

## How to Reproduce

1. Open `notebooks/ModelB_cGAN_Augmentation.ipynb` in [Google Colab](https://colab.research.google.com).
2. Set the runtime to GPU: **Runtime → Change runtime type → T4 GPU**.
3. Obtain a `kaggle.json` API token (kaggle.com → Account → Create New API Token) and upload it when
   prompted.
4. Run all cells (**Runtime → Run all**). Total runtime ≈ 1–2 hours.
5. Download `outputs.zip` at the end and unzip it into this repository.

To run locally instead of Colab:

```bash
pip install -r requirements.txt
jupyter notebook notebooks/ModelB_cGAN_Augmentation.ipynb
```

A CUDA-capable GPU is strongly recommended for cGAN training.
